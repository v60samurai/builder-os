# Discovery Templates

Two files, used in order, before any PRD gets opened.

| File | Use when |
|------|----------|
| [`idea-log.md`](./idea-log.md) | You have a raw idea and haven't pressure-tested it yet — BRAINSTORM mode |
| [`discovery-brief.md`](./discovery-brief.md) | An idea from the log is worth pressure-testing — DISCOVER mode |

## The rule both enforce

No solutioning here. If you catch yourself designing a feature instead of describing a problem, that thought belongs in a PRD draft, not in these two files.

## Exit condition

An idea only graduates to `prd/prd.md` once the discovery brief has a named riskiest assumption with a test plan, a structural (not demographic) ICP, and an honest steelman of the case against building it. See [`../pro/MANIFESTO.md`](../pro/MANIFESTO.md) for the full exit criteria.
