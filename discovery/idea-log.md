# Idea Log

> BRAINSTORM mode. Divergence only — no picking, no estimating, no killing ideas here. That happens in `discovery-brief.md`, one layer down.

The failure mode this file prevents: an idea gets talked out of existence in a Slack thread before it's ever written down next to its alternatives. Every idea gets a row, even the bad ones — especially the bad ones, because a bad idea written next to three others is what makes the good one visible.

**Rule:** log the idea the day it occurs to you. Don't wait until you've "thought it through" — half-formed is the point of this stage.

| Date | Idea (one sentence) | Who's it for | What problem, roughly | Kill / Keep / Promoted to Discovery | Notes |
|---|---|---|---|---|---|
| [YYYY-MM-DD] | [The idea, stated as a sentence a stranger could understand] | [User/segment, even a rough guess] | [The itch, in one line] | Keep | [Why it's still alive] |
| | | | | | |

**Promoted to Discovery** means the idea graduated — copy it into `discovery-brief.md` and start pressure-testing it. Leave the row here for the record; don't delete history.

**Kill** doesn't mean delete the row either. A killed idea with a reason attached is worth more than a re-litigated argument six months later when someone suggests it again.

### Exit criteria for this stage

You have enough when you have 3+ distinct ideas worth narrowing, not when you have one idea you've talked yourself into. If you only ever have one row "Keep," you skipped divergence — go back and generate two more before you move to Discovery.
