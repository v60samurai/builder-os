# Postmortem Template

One file, used after launch — LEARN mode.

[`postmortem-template.md`](./postmortem-template.md) measures actual results against the Success Criteria table you wrote in the PRD, extracts exactly one lesson (written as a rule, not an anecdote), and forces an explicit Iterate / Scale / Retire call.

## Schedule it now

The template's own instructions say this directly: put a date on the calendar for the postmortem when you set the launch date, not after. The most common failure isn't writing this badly — it's never writing it because everyone's already on the next fire by launch+1 week.

## Why this exists

A PRD without a postmortem is a hypothesis that never gets checked. See [`../pro/MANIFESTO.md`](../pro/MANIFESTO.md) — "LEARN mode is skipped and the team starts BRAINSTORM on the next bet" is a named tripwire, not a hypothetical.
